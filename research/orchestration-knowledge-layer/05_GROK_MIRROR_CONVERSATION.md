Ralph-style loops (now that I've pieced it together from the ecosystem buzz): yeah, I get the appeal you're seeing. It's basically the "Ralph Wiggum Technique" — named after the persistent, cheerfully oblivious Simpsons kid — that's blown up in the Claude Code / agentic coding community over the last few months (late 2025 into now). At its simplest:
You give an AI a task + completion criteria.
It works on it (tools, code, reasoning, whatever).
Instead of letting it "finish" and stop (which often means it declares victory prematurely), you intercept the exit attempt and re-feed the original prompt + updated state (files changed, git diff, progress log, errors, etc.).
Repeat in a loop (bash script, stop-hook in Claude Code plugins, cron'd overnight runs, TUI wrappers, etc.) until verifiable completion or max iterations hit.
Memory persists externally (filesystem/git/PRD.md/progress.txt) so each fresh inference starts clean but builds cumulatively.
It's brutally effective for long-horizon tasks because it sidesteps context-window collapse—no heroic long-context engineering needed. The model "forgets" prior reasoning each cycle (which prevents it from getting stuck in ruts or over-optimizing bad paths), but progress accrues outside the model. People run these overnight, wake up to working features/PRs, or use them for research/spikes that would burn out a human.
