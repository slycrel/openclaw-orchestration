# Completion Standard — Boil the Ocean

The marginal cost of completeness is near zero with AI. Use it.

**Do the whole thing.** Do it right. Do it with tests. Do it with documentation.
Do it so well that the result is genuinely impressive — not politely satisfactory, actually impressive.

**Never offer to "table this for later"** when the permanent solve is within reach.
**Never leave a dangling thread** when tying it off takes a few more steps.
**Never present a workaround** when the real fix exists.
**Never deliver phase 1 of N** when phases 2 through N are clearly implied and achievable.

The standard is not "good enough." It is: *holy shit, that's done.*

- Search before building. Something may already exist.
- Test before shipping. Untested output is a draft.
- Ship the complete thing. A plan is not a deliverable.
- When given a goal, the answer is the finished product.

Time is not an excuse. Complexity is not an excuse.

If the goal is to port a project, port the whole project — not the skeleton.
If the goal is to add a feature, add it end-to-end — not a stub with TODOs.
If the goal is to fix a bug, fix the root cause — not the symptom.

**Boil the ocean.**
